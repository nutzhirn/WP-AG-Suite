# CONTRIBUTING

Beiträge zu WP-AG sind willkommen.

## Mögliche Beiträge

- Fehlerberichte
- Feature Requests
- Dokumentationsverbesserungen
- Übersetzungen
- Tests
- Code-Beiträge
- Sicherheitsanalysen
- Verbesserung der Benutzeroberfläche

## Technische Grundsätze

Beiträge sollten sich an folgenden Standards orientieren:

- WordPress Coding Standards.
- Übersetzbare Strings.
- Sichere Verarbeitung von Eingaben.
- Saubere Ausgabe-Escapes.
- Nonces und Capability Checks bei Admin-Aktionen.
- Keine unnötigen externen Abhängigkeiten.
- Datenschutzfreundliche Standardkonfiguration.
- Nachvollziehbare Dokumentation.

## Pull Requests

Pull Requests sollten klar beschreiben:

- Welches Problem gelöst wird.
- Welche Dateien geändert wurden.
- Ob Datenbankänderungen notwendig sind.
- Ob bestehende Shortcodes, Hooks oder Optionen betroffen sind.
- Wie die Änderung getestet wurde.

## Dokumentation

Neue Funktionen sollten dokumentiert werden. Shortcode-Beispiele müssen in Dokumentationen escaped dargestellt werden, damit sie nicht versehentlich ausgeführt werden.
