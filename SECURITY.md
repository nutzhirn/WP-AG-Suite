# SECURITY

## Sicherheitsmeldungen

Sicherheitsprobleme sollten nicht zuerst öffentlich als GitHub Issue gemeldet werden.

Bitte Sicherheitsprobleme vertraulich melden, damit sie geprüft und behoben werden können, bevor Details öffentlich dokumentiert werden.

## Erwartete Sicherheitsstandards

WP-AG soll sich an den WordPress-Sicherheitsstandards orientieren:

- Sanitizing von Eingaben.
- Escaping von Ausgaben.
- Nonces für Admin-Aktionen.
- Capability Checks.
- Prepared SQL Statements.
- Keine unnötige Offenlegung sensibler Serverinformationen.
- Datenschutzfreundliche Standardwerte.
- Vorsicht bei Export- und Importfunktionen.

## Unterstützte Versionen

Während der frühen Entwicklungsphase gilt grundsätzlich die aktuellste veröffentlichte Version als unterstützte Version.
